# Demo Webshop E2E Automation (Selenium + TestNG + Cucumber)

This project automates an end-to-end purchase flow on:
https://demowebshop.tricentis.com

## Tech Stack
- Java 17
- Selenium WebDriver
- TestNG
- Cucumber (BDD)
- WebDriverManager
- SLF4J Simple Logger
- Maven

## AI Automation Tools Used
- GitHub Copilot
- Codex
- GitHub Copilot CLI 
- Codex CLI
- Playwright MCP Server
- Gemini CLI

## Project Structure
- `src/test/java/tests` - TestNG end-to-end test code
- `src/test/java/pages` - Page Object Model classes
- `src/test/java/stepdefinitions` - Cucumber step definitions
- `src/test/java/runners` - Cucumber TestNG runner
- `src/test/java/utils` - Driver, waits, and config helpers
- `src/test/resources/features` - Feature files
- `src/test/resources/config.properties` - URL, credentials, and test data

## Covered Validations
- Login success
- Product added to cart
- Cart item matches selected product
- Order success confirmation
- Logout success

## Prerequisites
- Java 17+
- Maven
- Google Chrome

## Configuration
Create a local `src/test/resources/config.properties` file before running the tests.
You can copy the values from `src/test/resources/config.example.properties` and replace the placeholder credentials and API key with your own.

## How To Run
From the project root, use one of the following commands.

### Run the full suite
This runs everything listed in `testng.xml`, which currently includes the TestNG E2E test and the Cucumber runner.

```bash
mvn clean test
```

### Run only the Cucumber feature
This runs the Cucumber TestNG runner only.

```bash
mvn -Dtest=runners.CucumberTestRunner test
```

### Run only the TestNG E2E test
This runs the page-object-based TestNG test only.

```bash
mvn -Dtest=tests.DemoWebShopE2ETest test
```

### Run in headless mode
This is useful for CI or when you do not want the browser UI to open.

```bash
mvn -Dheadless=true clean test
```

### Run only the Cucumber feature in headless mode

```bash
mvn -Dheadless=true -Dtest=runners.CucumberTestRunner testing
```

## Notes
- Browser setup uses WebDriverManager.
- Default browser is Chrome (set in `config.properties`).
- `headless` can be controlled through `config.properties` or overridden from the command line using `-Dheadless=true`.
- Test reports are generated by TestNG and Cucumber under `target/` named as "cucmber-report.html" files. One more report is under surefire-reports/index.html# that is also under `target/` folder.
